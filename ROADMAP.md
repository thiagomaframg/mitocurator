## Current implemented run order

The current `mitocurator run` workflow follows the evidence-driven order below:

1. Tool check
2. Initial annotation with MitoFinder, when the input is FASTA
3. Annotation refinement
4. Rotation
5. Diagnosis and annotation QC
6. Annotation assessment report
7. Read mapping to the current mitogenome
8. Variant evidence from read-mapping BAMs
9. Read-support interpretation of problematic stop codons
10. Targeted read extraction
11. Reconstruction pools
12. Targeted consensus
13. Candidate assembly
14. Integrated final report

This order separates evidence generation from evidence interpretation:

- `06_annotation_assessment/` summarizes annotation-level problems and review targets.
- `08_read_mapping/` generates BAM files and coverage summaries.
- `09_variant_evidence/` calls and summarizes SNP/indel evidence.
- `10_read_support/` interprets problematic CDS stop codons using the existing BAMs from `08_read_mapping/`.
- `11_targeted_extraction/` extracts reads for missing genes, problematic CDSs and correction candidates.
- `12_reconstruction_pools/` prepares targeted and mitogenome-mapped read pools.
- `13_targeted_consensus/` evaluates targeted consensus support for candidate genes or correction regions.
- `15_integrated_report/` integrates all evidence into gene-level curation decisions.

Therefore, `read-support` no longer performs read mapping by default. It reuses the alignment evidence generated by the dedicated read-mapping stage.

## Integrated final report

The current pipeline includes a final integrated reporting step:

    15_integrated_report/
    ├── integrated_curation_report.md
    ├── integrated_curation_report.html
    ├── integrated_gene_decisions.tsv
    └── evidence_index.tsv

This report integrates evidence from:

- `06_annotation_assessment/`
- `08_read_mapping/`
- `09_variant_evidence/`
- `10_read_support/`
- `13_targeted_consensus/`

The report summarizes each gene or feature into an integrated curation decision, including:

- `NO_ACTION_SUPPORTED`
- `STOP_CONFIRMED_BY_READS`
- `SEQUENCE_CORRECTION_SUPPORTED_BY_ALL_READSETS`
- `MISSING_GENE_CANDIDATE_SUPPORTED_ACROSS_READSETS`
- `MISSING_OR_INCOMPLETE_UNRESOLVED`

The report can be generated independently with:

    python -m mitocurator.cli integrated-report --config config.yaml

It is also executed as the final step of:

    python -m mitocurator.cli run --config config.yaml

The integrated report is diagnostic and decision-support oriented. It does not automatically modify the GenBank file.
