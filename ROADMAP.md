## Current implemented run order

The current `mitocurator run` workflow follows the evidence-driven order below:

1. Tool check
2. Initial annotation with MitoFinder, when the input is FASTA
3. Annotation refinement
4. Rotation
5. Diagnosis and annotation QC
6. Annotation assessment report
7. Read mapping to the current mitogenome
8. Variant evidence from read-mapping BAMs
9. Read-support interpretation of problematic stop codons
10. Targeted read extraction
11. Reconstruction pools
12. Targeted consensus
13. Candidate assembly

This order separates evidence generation from evidence interpretation:

- `08_read_mapping/` generates BAM files and coverage summaries.
- `09_variant_evidence/` calls and summarizes SNP/indel evidence.
- `10_read_support/` interprets problematic CDS stop codons using the existing BAMs from `08_read_mapping/`.

Therefore, `read-support` no longer performs read mapping by default. It reuses the alignment evidence generated by the dedicated read-mapping stage.

A future cleanup should renumber output directories to match the implemented execution order.
